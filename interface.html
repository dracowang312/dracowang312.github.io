  1. 访问权限修饰符的作用及区别：

    • public：任何地方都可以访问。
    • private：仅限于类的内部访问。
    • protected：限于类内部及其子类访问。
    • internal：限于同一程序集内的访问。
    • protected internal：限于同一程序集内和派生类访问。

  2. override与重载(overload)的区别：

    • override：重写基类中的虚方法，需要使用override关键字。
    • overload：同一个类中方法名相同，但参数列表不同。

  3. C#中索引器的特点：

    • 允许对象像数组一样通过索引访问。
    • 通过this关键字实现。
    • 可以重载，支持多维索引。

  4. 类和结构体的区别：

    • 类：引用类型，存储在堆上，支持继承。
    • 结构体：值类型，存储在栈上，不支持继承。

  1. 装箱(boxing)和拆箱(unboxing)的概念及区别：

    • 装箱：将值类型转换为引用类型（object）。
    • 拆箱：将引用类型转换回值类型。

  2. 堆和栈的区别：

    • 栈：存储值类型和方法调用信息，速度快，自动管理。
    • 堆：存储引用类型和动态分配的对象，速度较慢，需手动管理。

  3. foreach循环的对象要求：

    • 必须实现IEnumerable或IEnumerator接口。

  4. string对象的创建方式：

    • 通过字面量赋值：string str = "hello";
    • 通过构造函数：string str = new string('c', 5);

  5. 面向过程与面向对象的区别：

    • 面向过程：以过程（函数）为中心，注重步骤。
    • 面向对象：以对象为中心，注重封装、继承和多态。

  1. 请简述访问数据库时引入事务管理机制的作用：

    • 事务管理：确保数据库操作的原子性、一致性、隔离性和持久性（ACID特性）。
      • 原子性：事务中的所有操作要么全部成功，要么全部失败。
      • 一致性：事务使数据库从一种一致状态变为另一种一致状态。
      • 隔离性：并发事务之间互不影响。
      • 持久性：事务完成后对数据库的改变是永久的。

  2. 请简述inner join、left join、right join的区别：

    • INNER JOIN：返回两个表中匹配的记录。
    • LEFT JOIN：返回左表中的所有记录及右表中匹配的记录，没有匹配的部分显示NULL。
    • RIGHT JOIN：返回右表中的所有记录及左表中匹配的记录，没有匹配的部分显示NULL。

  3. **索引的作用?它的优缺点是什么?**：

    • 作用：加快数据检索速度。
    • 优点：提高查询效率。
    • 缺点：占用磁盘空间，增加插入、删除和更新操作的时间。
