(day01)

1. 访问权限修饰符的作用及区别：

  • private：只能在声明它的类或结构体内部访问。
  • protected：只能在声明它的类或其子类中访问。
  • public：可以从任何位置访问。
  • internal：只能在相同程序集中访问。

2. override与重载(overload)的区别：

  • override：用于子类重写父类的虚方法。
  • **重载(overload)**：在同一类中，可以有多个同名方法，但参数类型或个数必须不同。

3. C#中索引器的特点：

  • 可以根据数字以外的类型进行索引。
  • 允许多个索引器参数。

4. 类和结构体的区别：

  • 类是引用类型，结构体是值类型。
  • 类支持继承，结构体不支持。
  • 类在堆上分配内存，而结构体在栈上分配内存。

1. 装箱(boxing)和拆箱(unboxing)的概念及区别：

  • **装箱(boxing)**：将值类型转换为引用类型。
  • **拆箱(unboxing)**：将引用类型转换为值类型。

2. 堆和栈的区别：

  • 堆用于动态分配内存，由CLR进行垃圾回收。
  • 栈用于静态分配内存，存放方法调用、局部变量等。

3. foreach循环的对象要求：

  • 对象需实现IEnumerable或IEnumerable接口。

4. string对象的创建方式：

  • string s = new string(“xyz”); 创建了两个string object。

5. 面向过程与面向对象的区别：

  • 面向过程将问题分解为一系列步骤。
  • 面向对象将问题看作一系列对象的交互。

(day05)

1. .NET中类和结构的区别：

  • 类是引用类型，结构是值类型。
  • 类可以继承，结构不支持继承。
  • 类在堆上分配内存，结构在栈上分配内存。

2. 接口与抽象类的关系：

  • 接口可以继承接口，抽象类可以实现接口，但抽象类不能继承实体类。
  • 接口和抽象类都是用来定义一组规范，但抽象类可以包含字段和实现细节，而接口只能定义成员。

3. 线程进入一个对象方法后的情况：

  • 如果方法没有被标记为synchronized，则其他线程可以同时进入该对象的方法。
  • 如果方法被标记为synchronized，则其他线程需要等待当前线程执行完该方法后才能进入。

4. C#是否可以对内存直接进行操作：

  • 可以使用unsafe代码块和指针来对内存进行直接操作，但这样做需要谨慎处理，因为不安全代码可能导致内存泄漏或其他问题。

5. 接口和抽象类的相同点和不同点：

  • 相同点：都可以定义一组规范，用于约束子类或实现类的行为。
  • 不同点：抽象类可以包含字段和实现细节，而接口只能定义成员，不能包含字段或实现细节。

(day06)

1. C#中静态成员和非静态成员的区别：

  • 静

态成员属于类，只会在内存中存在一份拷贝，而非静态成员属于实例，每个实例都有一份拷贝。

  • 静态成员可以通过类名直接访问，而非静态成员需要先创建类的实例才能访问。

2. 在C#中堆上分配数据的情况：

  • 当使用new运算符创建一个对象时，对象会在堆上分配内存。
  • 当使用装箱操作将值类型转换为引用类型时，会在堆上分配内存。

3. 方法覆盖(override)和方法重写(overriding)的区别：

  • 方法覆盖是子类重写父类的虚方法，需要使用override关键字。
  • 方法重写是子类隐藏父类的非虚方法，不需要使用override关键字。

4. 虚方法和抽象方法的区别：

  • 虚方法在父类中有默认实现，子类可以选择是否重写。
  • 抽象方法没有默认实现，子类必须重写。

5. 关键字ref、out、params的作用：

  • ref：用于将参数按引用传递给方法，可以在方法内修改参数的值。
  • out：与ref类似，但要求在方法内对参数进行初始化。
  • params：允许方法接受可变数量的参数，参数被封装为数组。

(day07)

1. New关键字的用法：

  • New关键字有多种用法，包括在类中创建对象实例、调用构造函数、隐藏基类成员等。

2. 索引器的实现过程及限制：

  • 索引器可以根据数字以外的类型进行索引，但其实现过程必须使用this关键字，并且至少有一个参数。
  • 索引器可以重载，但必须保证不同的索引器参数列表。

3. C#中面向对象的三大特性：

  • 封装：将数据和操作数据的方法打包到一个单元中，隐藏了数据的具体实现细节。
  • 继承：子类可以继承父类的字段和方法，并可以通过重写方法来修改或扩展父类的行为。
  • 多态：允许同一操作作用于不同的对象上，并且具有不同的行为。

4. 构造器(Constructor)的继承和覆盖：

  • 构造器可以被继承，但不能被继承的同时被重写。
  • 构造器不可以被覆盖，因为子类会自动调用父类的构造器。

5. 方法覆盖(override)和方法重写(overriding)的区别：

  • 方法覆盖是子类重写父类的虚方法，需要使用override关键字。
  • 方法重写是子类隐藏父类的非虚方法，不需要使用override关键字。

(day09)

1. 接口与类的关系：

  • 接口定义了一组规范，类实现了这些规范。
  • 类可以实现多个接口，但只能继承一个类。

2. ArrayList与List<string>的区别：

  • ArrayList是动态数组，可以存储任意类型的元素。
  • List<string>是泛型集合，只能存储字符串类型的元素。

3. Static关键字的用途：

  • 用于定义静态成员，静态成员属于类而不属于对象。
  • 静态方法可以直接通过类名调用，而不需要创建类的实例。

4. try-catch语句的作用：

  • 用于捕获和处理异常，防止程序崩溃。

5. 关键字”base”的作用及使用场景：

  • “base”关键字用于调用父类的构造函数或成员。
  • 可以在子类中使用”base”关键字调用父类的构造函数，执行父类的初始化工作。
